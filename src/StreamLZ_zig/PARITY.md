# StreamLZ Zig port — full-parity punch list

Goal: 100% parity between `src/StreamLZ_zig/` and `src/StreamLZ/` — no skipped branches, no deferred stubs, no hidden API differences. The list below was produced by manual audit on 2026-04-13. All gaps must be closed before the port is considered complete.

**How to apply:** When picking up the port, start from the first unchecked item in the "execution order" section. Each item lists its file/line citations. When an item lands, mark `[x]` and move on. Do not reorder without user permission.

This enforces the "no cross-group LZ refs" SC invariant by construction: matches discovered against `group_src` can only reach other bytes inside the same slice. Within a group, 4 chunks are compressed sequentially with group-relative `start_pos` so cross-chunk refs within the group are still allowed. Per-worker arena (step 14) is reset between groups so the per-group MLS + match-finder working set is released before the next group. New dispatch order in `compressFramedHigh`: (1) try SC parallel (skips the global MLS allocation entirely), (2) non-SC parallel, (3) serial. New roundtrips: L6 2 MB (2 groups, 2 threads), L7 1 MB (1 group), L8 2 MB (BT4 + 2 groups), L6 384 KB (partial last group). 270/270 unit tests pass.
- [x] **39. D5** — Multi-piece concatenated frame support on both encoder and decoder. **Encoder:** renamed the single-shot `compressFramed` body to `compressFramedOne` (internal), added a new public `compressFramed` wrapper that retries on `error.OutOfMemory` by stepping down a fallback ladder — 1 GB → 512 MB → 256 MB → 128 MB → 64 MB → 32 MB → 16 MB min piece — and compressing as N self-contained pieces at the smaller size. If even the 16 MB min-piece path OOMs, the error propagates. **Decoder:** split `decompressFramedInner` into an outer piece loop + inner `decompressOneFrame` that walks concatenated SLZ1 frames (each piece carries its own frame header + end mark). Single-frame inputs still decode in one iteration of the outer loop so there's no behavioral change for non-retry callers. Matches C# `DecompressCore` at `StreamLzDecoder.cs:482-548`. New tests: 64 KB split into two L1 SC pieces, and 64 KB split as one L6 + one L9 piece (verifies codec-switching across piece boundaries). Triggering the actual OOM ladder requires > 1 GB input which isn't practical in unit tests; the structural contract is the wire-format piece. 272/272 unit tests pass.
